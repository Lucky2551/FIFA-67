# ⚽ เขาวงกตหน้าเอียง (Face Tilt Maze — Football Edition)

เกมกลิ้ง "ลูกฟุตบอลจริง" ผ่านเขาวงกตในสนามหญ้า แบบสุ่มใหม่ทุกด่าน โดยควบคุมด้วยการ **เอียง/ขยับหน้า** หน้ากล้องเว็บแคม (รองรับ [Iriun Webcam](https://iriun.com/) ที่ใช้มือถือเป็นกล้อง) — พาลูกบอลกลิ้งไปให้ถึงประตู

ใช้ [MediaPipe Face Landmarker](https://ai.google.dev/edge/mediapipe/solutions/vision/face_landmarker) ผ่าน CDN เพื่อหาตำแหน่งจุดบนใบหน้า (ไม่มีการอัปโหลดวิดีโอไปเซิร์ฟเวอร์ไหน ประมวลผลในเบราว์เซอร์ทั้งหมด)

ไฟล์เดียวจบ (`index.html`) — รูปลูกบอล/ประตู/สนามถูกฝังเป็น base64 ไว้ในไฟล์เรียบร้อยแล้ว ไม่ต้องพกโฟลเดอร์รูปแยกไปด้วย push ขึ้น GitHub แค่ไฟล์เดียวก็ใช้ได้ทันที

## วิธีเล่น

1. เปิด Iriun Webcam บนมือถือ + โปรแกรมบนคอม เชื่อมต่อกันให้เรียบร้อย
2. เปิดไฟล์ `index.html` ผ่าน **https:// หรือ localhost เท่านั้น** (ดูหัวข้อด้านล่าง) — ต้องมีอินเทอร์เน็ตด้วย เพราะโมเดลตรวจจับใบหน้าโหลดจาก CDN ตอนเริ่มเกม
3. กด **เริ่มเกม** → อนุญาตสิทธิ์กล้อง → เลือกกล้อง Iriun Webcam ถ้ามีหลายตัว
4. รอโหลดโมเดล (ครั้งแรกอาจใช้เวลาสักครู่) จากนั้นระบบจะนับถอยหลังให้ **มองตรงกล้อง อยู่นิ่งๆ** เพื่อจับตำแหน่ง "จุดกึ่งกลาง" ของหน้าคุณ
5. หลังจากนั้นเอียง/ขยับหน้าไปทิศทางที่ต้องการให้ลูกบอลกลิ้งไป (ซ้าย ขวา บน ล่าง) เพื่อพาลูกบอลไปถึงธงสีแดง
6. ถึงธง → ขึ้นด่านใหม่ที่เขาวงกตใหญ่ขึ้นและซับซ้อนขึ้นอัตโนมัติ

ถ้าควบคุมรู้สึก "กลับด้าน" (เอียงซ้ายแต่บอลไปขวา หรือก้มหน้าแต่บอลขึ้น) ให้ติ๊กช่อง **สลับซ้าย-ขวา** / **สลับบน-ล่าง** เอาตามที่รู้สึกถูกต้อง เพราะทิศทางจริงขึ้นอยู่กับตำแหน่งวางกล้อง/มือถือของแต่ละคน

## รันเว็บอย่างไร

เบราว์เซอร์อนุญาตให้ใช้กล้องเฉพาะหน้าเว็บที่เป็น HTTPS หรือ localhost เท่านั้น

### วิธีที่ 1 — GitHub Pages (แนะนำ)

1. Push โฟลเดอร์นี้ขึ้น repository บน GitHub
2. Settings → Pages → เลือก branch/root ที่มีไฟล์
3. เปิดลิงก์ `https://<username>.github.io/<repo>/` บนคอมที่ต่อกับ Iriun Webcam ไว้

### วิธีที่ 2 — รันในเครื่อง

```bash
python3 -m http.server 8000
```

แล้วเปิด `http://localhost:8000`

## เปลี่ยนรูปลูกบอล/ประตู/สนามเอง

รูปทั้งสามถูกฝังเป็น base64 ไว้ในตัวแปร `BALL_DATA_URI`, `GOAL_DATA_URI`, `FIELD_DATA_URI` ใกล้ๆ ฟังก์ชัน `loadImage` ในไฟล์ `index.html` ถ้าอยากเปลี่ยนรูป:

1. แปลงรูปใหม่เป็น base64 data URI (เช่น เว็บ [base64-image.de](https://www.base64-image.de/) หรือคำสั่ง `base64 -w0 รูปของคุณ.png`)
2. เอาข้อความที่ได้ (ขึ้นต้นด้วย `data:image/...;base64,...`) ไปแทนค่าตัวแปรเดิม
3. แนะนำให้ใช้รูปพื้นหลังโปร่งใส (PNG ตัดพื้นหลังออกแล้ว) สำหรับลูกบอลและประตู จะได้ดูเนียนกับพื้นสนาม และควรย่อขนาดรูปให้เล็กพอสมควรก่อน (ไม่เกินสองสามร้อยพิกเซล) ไม่งั้นไฟล์ HTML จะใหญ่และโหลดช้า

## หลักการทำงาน

1. **พื้นหลังหน้าเว็บ**: รูปสนามหญ้าถูกตั้งเป็นพื้นหลังของทั้งหน้าเว็บ (ชั้นหลังสุด อยู่ด้านหลังกล่องเกม) ไม่ได้วาดซ้ำอยู่ในตัวเขาวงกต — ทำให้กำแพง/ลูกบอลในกระดานดูชัดเจน ไม่แข่งกับลวดลายสนามด้านหลัง
2. **ตรวจจับหน้า**: ใช้ MediaPipe Face Landmarker วิเคราะห์วิดีโอแบบเรียลไทม์ ได้ตำแหน่ง landmark 468 จุดบนใบหน้า
3. **ตำแหน่งอ้างอิง**: ใช้จุดปลายจมูก (landmark index 1) เป็นตัวแทนตำแหน่งหน้า และระยะห่างระหว่างแก้มซ้าย-ขวา (index 234, 454) เป็นตัวปรับสเกล ไม่ให้ผลลัพธ์เปลี่ยนไปตามระยะใกล้-ไกลจากกล้อง
4. **คาลิเบรต**: ตอนเริ่มเกมจะเก็บค่าเฉลี่ยตำแหน่งจมูกตอน "มองตรง" ไว้เป็นจุดศูนย์ (neutral position)
5. **แปลงเป็นแรง**: ทุกเฟรมคำนวณระยะห่างของตำแหน่งจมูกปัจจุบันจากจุดศูนย์ แปลงเป็นค่าแรงเอียง (-1 ถึง 1) แล้วใช้เป็นความเร่งของลูกบอลบนกระดาน (เหมือนแรงโน้มถ่วงตอนเอียงกระดานไม้จริง)
6. **เขาวงกต**: สร้างด้วยอัลกอริทึม recursive backtracker (สุ่มทุกครั้งที่เริ่ม/ขึ้นด่านใหม่) แล้วเจาะทางเปิดเพิ่มอีกเล็กน้อยให้ไม่ยากเกินไป
7. **ฟิสิกส์**: ลูกบอลเคลื่อนที่ต่อเนื่อง (ไม่ใช่ขยับทีละช่อง) ชนกำแพงแล้วสะท้อน/หยุดตามปกติ

## ปรับแต่งเกม

เปิด `index.html` แล้วแก้ค่าตรงนี้ได้:

- `SENSITIVITY` — ความไวของการแปลงตำแหน่งหน้าเป็นแรงเอียง (ค่าสูง = ขยับหน้านิดเดียวก็แรงมาก)
- `ACCEL`, `FRICTION`, `MAX_SPEED` — ความรู้สึก "หนัก/เบา" ของลูกบอล
- `COLS`, `ROWS` ใน `resetProgress()` / `nextLevel()` — ขนาดเขาวงกตเริ่มต้นและอัตราการเพิ่มความยากต่อด่าน
- `extra` ใน `generateMaze()` — จำนวนทางลัดที่เจาะเพิ่ม ปรับให้เขาวงกตง่าย/ยากขึ้น

## ข้อควรรู้

- โมเดลและไลบรารีโหลดจาก CDN ของ Google/jsDelivr ตอนรันจริง หากอินเทอร์เน็ตช้าหรือถูกบล็อกโดเมนเหล่านี้ เกมจะโหลดไม่ขึ้น:
  - `cdn.jsdelivr.net`
  - `storage.googleapis.com`
- ทำงานได้ดีที่สุดในที่แสงสว่างเพียงพอ และให้หน้าคุณอยู่ในเฟรมกล้องตลอดเวลา
- ไม่มีการส่งภาพ/วิดีโอออกไปนอกเครื่อง ประมวลผลทั้งหมดในเบราว์เซอร์ (client-side)

## ต่อยอดได้

- ใช้ `outputFacialTransformationMatrixes` ของ MediaPipe เพื่อคำนวณมุมเอียงจริง (pitch/roll/yaw) แทนตำแหน่งจมูก จะได้ฟีลลิ่งเหมือน "เอียงหน้า" มากขึ้น
- เพิ่มหลุมพราง (ตกแล้วเริ่มใหม่ที่จุดเริ่มต้น) แบบเขาวงกตไม้จริง
- เพิ่มเสียงลูกบอลกลิ้ง/ชนกำแพง และเสียงฉลองตอนถึงธง
- บันทึกเวลาที่ดีที่สุดของแต่ละด่านด้วย `localStorage`
